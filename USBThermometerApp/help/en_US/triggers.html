Triggers

USB Thermometer

Help

Triggers are functions. Those functions are executed when temperature overs a specified threshold. USB Thermometer application has six different types of triggers:
  • Email notification trigger Sends an email when the trigger occurs.
  • System notification trigger Displays notification in system tray or plays a sound.
  • RS-232 trigger Changes state of serial port pins and/or sends a string of bytes (command) to a remote device.
  • TCP/IP and UDP trigger Sends a string of bytes (command) to a remote computer or device using specified internet protocol.
  • User trigger Executes user program.

Adding a new trigger

To manage triggers open the triggers configuration window from menu bar or toolbar. Click the New button on the window form and choose the trigger type from the list with appears after you have pressed the button.

The state of triggers

The newly created trigger stays inactive. The triggers configuration window has a list of triggers on the left side. The every trigger has an icon with identifies its state:
  •   The inactive trigger.
  •   The active trigger.
After the trigger has been configured, it is indispensable to activate it. To do so check an Enabled checkbox on the top of the configuration window. Every change of the trigger state or configuration must be confirmed. Click the Save button on the bottom off the window to save the trigger configuration and activate it.

Basic settings

Before activate the trigger, it is indispensable to configure it. The basic settings (common for all trigger types) has been described below. Specifics triggers settings have been described in separated chapters.
  • Name Name of the trigger allows to distinguish it from the others on the triggers list.
  • Sensor Sensor with temperature changes executes the trigger.
  • Threshold The Trigger executes below or over the value with has been specified as the threshold.
  • Trigger below or over threshold The setting configures if the trigger executes below or over the value specified as the threshold.
  • Hysteresis Next execution of the trigger is delayed until the temperature achieve the value less or bigger (it depends on the Trigger below or over threshold parameter) of the threshold value plus the hysteresis value. The hysteresis minimum value is 0.1 and must be always positive.
Example The threshold is set to 20.0, execution is set over the threshold, the hysteresis is set to 2.0. Temperature grows from 15.0 to 22.0. When the temperature achieves 20.1 the trigger executes. Subsequently the temperature fails to 19.0. When the temperature is falling the trigger does not execute. After this the temperature grows again to 23.0. When the temperature is growing the trigger does not execute because the hysteresis value has not been achieved. After the temperature falls below the 18.0, the hysteresis value will be achieved and next grow of the temperature will execute the trigger.
