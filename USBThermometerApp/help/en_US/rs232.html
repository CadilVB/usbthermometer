RS232 trigger allows to change state of serial port pins and/or sends a string of bytes (command) to a remote device. It is possible to control one serial port with multiple triggers. Thanks to this user is allowed to build a thermostat with multiple sensors control.

To set up the trigger settings, aside from setting up the basic configuration (common for all types of the triggers, it is necessary to configure additional settings:
  • Com port Name of the serial port. Typical name of serial port in Windows operating is COMx. Where x is a number of the serial port.
  • Baud rate Number of bits per second. Operating speed of the serial port.
  • Bits Number of bits which are one byte.
  • Stop bits Number of wait bits between sending or receiving bytes.
  • Parity Adds one bit (at the end of each byte) which controls data validity.
  • RTS Pin Three states are possible
    • Do not change Does not change a pin state while trigger executes.
    • Set to '1' Sets high state while trigger executes.
    • Set to '0' Sets low state while trigger executes.
  • DTR Pin Three states are possible
    • Do not change Does not change a pin state while trigger executes.
    • Set to '1' Sets high state while trigger executes.
    • Set to '0' Sets low state while trigger executes.
  • Send a command Enables command sending while trigger executes.
  • Command Bytes to send. It is possible to edit message using two modes: ASCI or HEX. In ASCI mode command is typed using standard characters which are available on keyboard (letters, numbers, etc). In HEX mode it is possible to type any byte value by typing its hexadecimal value.
