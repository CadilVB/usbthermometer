Menu bar

The structure of the menu bar is as follows:
  • File
    •   New workspace Closes all opened internal windows and left blank desktop.
    •   Open workspace Opens previously saved workspace.
    •   Save workspace Saves current internal windows view and its settings. Notification! File default.utw saves internal windows setting which are restored when the application start. The file is overwritten when the application is closing.
    •   Close Closes the application.
  • Tools
    •   Graph Adds a new graph internal window to the workspace.
    •   Bar graph Adds a new bar graph internal window to the workspace.
    •   Table Adds a new table internal window to the workspace.
    •   Report Adds a new report internal window to the workspace.
    •   Alerts and triggers Opens a triggers configuration window.
    •   Configuration Opens a configuration window.
  • Window
    •   Tile Arranges internal windows in non-overlapping tiles.
    •   Tile horizontally Arranges internal windows in non-overlapping horizontal tiles.
    •   Tile vertically Arranges internal windows in non-overlapping vertical tiles.
    •   Cascade Arranges internal windows in an overlapped fashion.
    •   Internal windows list List of opened internal windows.
  • Help
    •   help Display help content you are reading now.
    •   About Displays the window contains information like version of the application, license, etc.

Toolbar

The toolbar contains some elements duplicated from the menu bar:
  • Configuration Opens a configuration window.
  • Graph Adds a new graph internal window to the workspace.
  • Bar graph Adds a new bar graph internal window to the workspace.
  • Table Adds a new table internal window to the workspace.
  • Report Adds a new report internal window to the workspace.
  • Alerts and trigger Opens a triggers configuration window.

Sensors list

On the left of the main application window is a list of sensors and devices connected to this computer. Elements are displayed in form of the tree as follows:
  •   Database Working database: [database type]:[database name].
  •   USB Thermometer devices Devices connected to the computer. The name of a device is its serial number.
  •   Active temperature sensor The icon of a working temperature sensors.
  •   Not active temperature sensor The sensor which is not connected to the computer at this moment. It is still possible to display the data collected by this sensor in the pass.

State bar

The state bar of the application contains information as follows: the name of a selected sensor, its ID number, the type of the sensor and the application working state.
